---
title: API Client
description: The createApi(fetcher) factory and how to use it.
---

`api.ts` exports a **factory**, Tuyau-style: you inject the fetcher at runtime instead
of the codegen hardcoding an import path. This is what makes the client portable — you
control the transport, base URL, auth, and serialization.

```ts title="src/lib/api.ts"
import { createApi } from '../generated/api';
import { createFetcher } from '@dudousxd/nestjs-client';

export const api = createApi(
  createFetcher({ baseUrl: '/api', headers: () => ({ authorization: token() }) }),
);
```

The result is a nested object keyed by route name (`controller.method`), fully typed
from your contracts.

## Reads vs writes

GET routes produce **`queryOptions`**; the rest produce **`mutationOptions`** — the
helpers from your TanStack adapter (see [TanStack Query](/docs/tanstack-query)).

```tsx
import { useQuery, useMutation } from '@tanstack/react-query';
import { api } from '../lib/api';

// GET /users  → queryOptions, data typed as User[]
const users = useQuery(api.users.list());

// GET /users/:id  → params are typed + required
const user = useQuery(api.users.show({ params: { id } }));

// POST /users  → mutationOptions; mutate({ body }) typed from the DTO
const create = useMutation(api.users.create());
create.mutate({ body: { email: 'a@b.com', password: 'secret123' } });
```

## Inferring types

The generated `Api` type and the per-route response/body types are available without
calling anything:

```ts
import type { Api } from '../generated/api';
// e.g. Awaited<ReturnType<typeof api.users.show>> for the queryOptions object
```

## Inertia mode

With `mutationClient: 'inertia'`, the generated client additionally exposes a typed
`navigate()` helper backed by the Inertia `router`, and mutations integrate with
Inertia visits. See [nestjs-inertia](/docs/inertia).
