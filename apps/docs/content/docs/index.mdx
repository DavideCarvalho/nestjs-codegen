`nestjs-codegen` reads your NestJS controllers, contracts, and DTOs and generates a
**fully-typed client**: a route map, a typed API client, and client-side validation
schemas. It works **with or without Inertia.js**, and every moving part is pluggable —
the validation library, the HTTP client, the serializer, and the query layer.

```ts title="src/app.module.ts"
import { Module } from '@nestjs/common';
import { NestjsCodegenModule } from '@dudousxd/nestjs-codegen/nest';

@Module({
  imports: [
    NestjsCodegenModule.forRoot({
      contracts: { glob: 'src/**/*.controller.ts' },
      codegen: { outDir: 'src/generated' },
    }),
  ],
})
export class AppModule {}
```

Import the module and the client regenerates with your dev server. For CI, the same
generator ships as a CLI — `npx nestjs-codegen codegen` — to fail the build before
deploy if the committed client drifts. See [Getting started](/docs/getting-started).

## What you get

The codegen writes these files into your output directory:

- **`routes.ts`** — a `ROUTES` map, a `RouteName` union, and a typed `route()` helper.
- **`api.ts`** — a Tuyau-style `createApi(fetcher)` factory; nested by route name, fully typed.
- **`forms.ts`** — a validation schema per validated endpoint, in your chosen lib.
- **`pages.d.ts` / `components.json`** — when the Inertia integration is enabled.

## Key features

- **Pluggable validation** — one neutral `SchemaNode` IR, rendered by [zod](https://zod.dev)
  (bundled), [valibot](https://valibot.dev), or [arktype](https://arktype.io) adapters.
- **Typed API client** — `createApi(fetcher)`: inject your own client at runtime.
- **Extension-based integrations** — register integrations in `extensions: [...]`; each
  shapes the generated `api.ts`. See [Extensions](/docs/extensions).
  - **TanStack Query** (`tanstackQuery()`) — emits `queryOptions`/`mutationOptions` from
    *your* adapter package (react/vue/svelte/solid); never forces `@tanstack/query-core`.
  - **nestjs-filter** (`nestjsFilterCodegen()`) — `@FilterFor`/`@ApplyFilter` → typed
    `filterQuery()` helpers.
  - **nestjs-inertia** (`nestjsInertiaCodegen()`) — pages, shared props, and the Inertia
    `router` + `navigate()` helper.
- **Bring your own fetcher** — native `fetch` by default, or an axios instance via
  `axiosTransport()`. superjson and transformer pipelines supported.
